Right now the viewer is treating the saved X/Y and font sizes as if they were *logical canvas* pixels, but in the iPhone design they‚Äôre actually *physical* pixels (multiplied by DPR ‚âà 3). So everything is coming out ~‚Öì size and pushed inwards.

We don‚Äôt need any more fancy detection ‚Äì we just need to **always divide inscription + motif data by the saved design DPR** (`shapeData.dpr`) before we apply `uniformScale`.

Below is a **minimal, targeted patch** you can drop into `DesignPageClient.tsx`.

### 1Ô∏è‚É£ Fix inscriptions: use DPR-normalised canvas coords

Search in `DesignPageClient.tsx` for the comment:

```ts

```

and replace the whole mapping block inside `renderInscriptions` with this:

```ts
// üîç INSCRIPTION FINAL mapping
const inscriptionElements = designInscriptions.map((inscription, index) => {
  const {
    displayWidth,
    displayHeight,
    offsetX,
    offsetY,
    initW,
    initH,
    uniformScale,
    designDpr,
  } = scalingFactors;

  // Design DPR (how the design was authored ‚Äì 1 for desktop, 2/3 for retina)
  const savedDpr = designDpr || 1;

  // ---- 1. Read raw saved values ----
  const rawX = typeof inscription.canvas_x === 'number' ? inscription.canvas_x : 0;
  const rawY = typeof inscription.canvas_y === 'number' ? inscription.canvas_y : 0;
  const rawFontSize =
    typeof inscription.font_size === 'number' ? inscription.font_size : 24;

  // ---- 2. Convert PHYSICAL (DPR-scaled) ‚Üí LOGICAL canvas coords ----
  // All inscriptions/motifs are saved in the same space as the screenshot:
  //   logical_canvas = physical_pixels / savedDpr
  const canvasX = rawX / savedDpr;
  const canvasY = rawY / savedDpr;
  const canvasFontSize = rawFontSize / savedDpr;

  // ---- 3. Canvas centre ‚Üí top-left, then apply uniformScale ----
  const displayX =
    offsetX + (canvasX + initW / 2) * uniformScale;
  const displayY =
    offsetY + (canvasY + initH / 2) * uniformScale;

  const fontSizePx = canvasFontSize * uniformScale;

  if (index === 0) {
    console.log('üìù INSCRIPTION FINAL mapping', {
      raw: { x: rawX, y: rawY, fontSize: rawFontSize, savedDpr },
      canvas: { x: canvasX, y: canvasY, fontSize: canvasFontSize },
      display: { x: displayX, y: displayY, fontSizePx },
      frame: { initW, initH, displayWidth, displayHeight, offsetX, offsetY, uniformScale },
    });
  }

  const inscriptionStyle: React.CSSProperties = {
    position: 'absolute',
    left: `${displayX}px`,
    top: `${displayY}px`,
    transform: 'translate(-50%, -50%)',
    fontFamily: inscription.font_family || 'Times New Roman, serif',
    fontSize: `${fontSizePx}px`,
    fontWeight: inscription.font_weight || 'normal',
    textAlign: 'center',
    whiteSpace: 'pre-line',
    color: 'black',
    lineHeight: 1.1,
    pointerEvents: 'auto',
  };

  return (
    <DraggableElement key={inscription.id ?? index} initialStyle={inscriptionStyle}>
      {inscription.text}
    </DraggableElement>
  );
});
```

This does exactly three things:

1. Takes the raw saved numbers.
2. Divides by `designDpr` to get the **authoring canvas** values.
3. Applies `uniformScale` + centre-origin conversion.

### 2Ô∏è‚É£ Fix motifs: same DPR normalisation

Now search for the motifs block with:

```ts

```

and inside the `map` callback replace the positioning + sizing part with:

```ts
const savedDpr = scalingFactors.designDpr || 1;

// ---- 1. Raw saved coords (physical) ----
const rawX = typeof motif.x === 'number' ? motif.x : 0;
const rawY = typeof motif.y === 'number' ? motif.y : 0;

// ---- 2. PHYSICAL ‚Üí LOGICAL canvas coords ----
const canvasX = rawX / savedDpr;
const canvasY = rawY / savedDpr;

// ---- 3. Canvas centre ‚Üí top-left, then uniformScale ----
const displayX =
  scalingFactors.offsetX +
  (canvasX + scalingFactors.initW / 2) * scalingFactors.uniformScale;
const displayY =
  scalingFactors.offsetY +
  (canvasY + scalingFactors.initH / 2) * scalingFactors.uniformScale;

// ---- 4. Motif height in canvas space (also normalised by DPR) ----
const motifRatio = parseFloat(motif.ratio || '1');
const motifSrc = motif.src || motif.name;
const svgDims = motifDimensions[motifSrc];

const canvasH = scalingFactors.initH || 600;
const uniformScale = scalingFactors.uniformScale || 1;

let motifHeightCanvas: number | undefined;
if (typeof motif.height === 'number') {
  motifHeightCanvas = motif.height / savedDpr;
}

const aspect =
  svgDims && svgDims.height > 0 ? svgDims.width / svgDims.height : 1;

// Final size in display pixels
let motifHeight = motifHeightCanvas
  ? motifHeightCanvas * uniformScale
  : (svgDims ? svgDims.height * motifRatio * uniformScale : 100 * uniformScale);

let motifWidth = motifHeightCanvas
  ? motifHeightCanvas * aspect * uniformScale
  : (svgDims ? svgDims.width * motifRatio * uniformScale : 100 * uniformScale);

// Guard rails
const minH = 24;
const maxH = canvasH * uniformScale * 0.5;
motifHeight = Math.min(Math.max(motifHeight, minH), maxH);
motifWidth = Math.max(motifWidth, minH * aspect);

if (index === 0) {
  console.log('üé® MOTIF FINAL mapping', {
    src: motifSrc,
    raw: { x: rawX, y: rawY, height: motif.height, savedDpr },
    canvas: { x: canvasX, y: canvasY, height: motifHeightCanvas },
    display: { x: displayX, y: displayY, width: motifWidth, height: motifHeight },
    frame: { initW: scalingFactors.initW, initH: scalingFactors.initH, uniformScale },
  });
}
```

Leave the `DraggableElement` and the inner `<img>/<svg>` rendering as they are ‚Äì just hook this new `displayX/displayY/motifWidth/motifHeight` into the `initialStyle`.

### What this should fix

* **SEHOVIC on iPhone DPR=3**
  Inscriptions + motifs will use the same logical 414√ó660 authoring canvas as the screenshot, so the layout and sizes will match the original editor instead of shrinking.

* **Old desktop designs**
  `designDpr` is 1, so dividing by 1 is a no-op ‚Äì they stay exactly as they are now.
