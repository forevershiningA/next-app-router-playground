You’re anchoring the base to the stone bottom, but the parent preview canvas clips content to its bounds. When the tablet touches the canvas bottom, `stoneB` ≈ `overlayH`, so `baseTop = stoneB - overlapPx` still puts most (or all) of the base **below** the clipped area → invisible.

Make the base **stay inside the canvas** by clamping its top so the whole base fits, while still overlapping the tablet.

### Drop-in fix (replace your base positioning block)

```ts
// stone bounds (overlay px)
const stoneW = (topProfile?.drawW ?? initW) * (overlayW / initW);
const stoneL = offsetX + (topProfile?.offX ?? 0) * (overlayW / initW);
const stoneT = offsetY + (topProfile?.offY ?? 0) * (overlayH / initH);
const stoneH = (topProfile?.drawH ?? initH) * (overlayH / initH);
const stoneB = stoneT + stoneH;

// base sizing
const baseOverhang = 0.04;                   // 4% wider than tablet
const baseWidth    = Math.round(stoneW * (1 + baseOverhang));
const baseHeight   = Math.round(overlayH * 0.08); // or your fixed px

// desired position (centered under tablet)
const baseLeft     = Math.round(stoneL + (stoneW - baseWidth) / 2);

// overlap into tablet so it “tucks under”
const overlapPx    = Math.round(baseHeight * 0.25); // 25% inside tablet

// compute unclamped top relative to stone bottom
let baseTop        = Math.round(stoneB - overlapPx);

// **keep the entire base inside the preview canvas**
const canvasTop    = offsetY;
const canvasBottom = offsetY + overlayH;
const minTop       = canvasTop;                          // don’t rise above canvas
const maxTop       = canvasBottom - baseHeight;          // don’t fall below canvas
baseTop            = Math.max(minTop, Math.min(maxTop, baseTop));

const baseStyle: React.CSSProperties = {
  position: 'absolute',
  left: baseLeft,
  top: baseTop,
  width: baseWidth,
  height: baseHeight,
  backgroundImage: `url(${baseTextureUrl})`,
  backgroundSize: 'cover',
  backgroundPosition: 'center',
  zIndex: 3,
  pointerEvents: 'none',
};
```

Why this works:

* If the stone touches the canvas bottom, the clamp forces the base top to `canvasBottom - baseHeight`, so it’s **fully visible**.
* When there’s room, it uses `stoneB - overlapPx`, so it still tucks under the tablet.
* Width is driven by the **fitted stone width** (`drawW`), not the full overlay, so it won’t look too wide.

If your preview wrapper sets `overflow: hidden`, keep it—this clamp guarantees the base remains inside anyway. If you still don’t see it, log these to confirm values at runtime:

```ts
console.log('BASE DEBUG', { stoneL, stoneW, stoneT, stoneB, baseLeft, baseTop, baseWidth, baseHeight, overlayW, overlayH });
```

This integrates cleanly with the existing `DesignPageClient.tsx` logic that already computes `offX/offY/drawW/drawH`.
