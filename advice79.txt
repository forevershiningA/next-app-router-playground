You’re right—the base should use the **base length from the JSON (mm)** and be only a bit wider than the headstone, **not** fill the whole screen.

Here’s the minimal, correct mapping and placement (already present in your file; keep this version and remove the full-width variant):

1. **Read mm from the design**

* `tabletWidthMm = headstoneItem.width`
* `lengthMm = baseItem.width || baseItem.length || (tabletWidthMm + 2*25)`
* `heightMm = baseItem.height || baseItem.thickness_mm || 90` 

2. **Convert mm → px using the same scale as the headstone**

* `pxPerMm = uniformScale * (initW / tabletWidthMm)`
* `baseWidthPx  = Math.round(lengthMm * pxPerMm)`
* `baseHeightPx = Math.round(heightMm * pxPerMm)` 

3. **Place and center the base under the headstone**

* Compute the tablet bottom: `stoneBot = Math.round(offsetY + (initH * uniformScale))`
* Make the row span the overlay width for centering, but give the base its **computed pixel width**:

```ts
const baseRowStyle = {
  position: 'absolute',
  left: 0,
  top: stoneBot,
  width: displayWidth,
  height: baseHeightPx,
  display: 'flex',
  justifyContent: 'center',
  alignItems: 'flex-start',
  pointerEvents: 'none',
  zIndex: 2,
};

const baseStyle = {
  width: baseWidthPx,
  height: baseHeightPx,
  backgroundImage: baseTextureData ? `url(${baseTextureData})` : 'linear-gradient(to bottom,#1a1a1a,#000)',
  backgroundSize: 'cover',
  backgroundPosition: 'center',
  backgroundColor: '#2b2b2b',
};
```

Then render `<div style={baseRowStyle}><div style={baseStyle} /></div>`. 

4. **Remove the full-width base renderer**
   You also have another block that forces the base to `width: '100%'` across `displayWidth`. Delete that block so it can’t override the mm-based sizing. It’s the one with comment “Span the whole preview width” and `baseStyle = { width: '100%' … }`. 

With the mm→px mapping above, the base width will exactly follow the JSON (typically a small overhang beyond the tablet), matching the original screenshot behavior.
