Yep—your base is still sized a bit “blind.” Two fixes will make it match the original JSON/screenshot exactly:

## 1) Use one isotropic px/mm (from tablet width)

Don’t derive Y from tablet height (it can drift). Use **px/mm = stoneW / tabletWidthMm** for both W and H.

```ts
// --- isotropic px/mm from the actually fitted tablet (width only)
const pxPerMm = stoneW / tabletWidthMm;
```

## 2) Read base length/height from the JSON (with smart fallbacks)

Designs store base dimensions under different keys. Pick the *largest plausible mm field* for length (to avoid `_length`=thickness) and only overhang if no value exists.

```ts
// find a base-like addition in the design JSON
const baseItem = (designData || []).find((it:any) =>
  /base/i.test(`${it.type} ${it.name ?? ''} ${it.label ?? ''}`)
) ?? {};

// collect candidate mm fields
const mmCandidates = [
  Number(baseItem.length_mm),
  Number(baseItem.length),
  Number(baseItem.width_mm),
  Number(baseItem.width),
  Number(baseItem.size?.length_mm),
  Number(baseItem.size?.width_mm),
];

// choose the biggest plausible length (avoid thickness fields like 60–120 mm)
const lengthMm = mmCandidates
  .filter(v => Number.isFinite(v) && v >= 300) // 300+ mm => likely a base length
  .sort((a,b) => b - a)[0];

const heightCandidates = [
  Number(baseItem.height_mm),
  Number(baseItem.height),
  Number(baseItem.size?.height_mm),
  Number(baseItem.thickness_mm),
];
const heightMm = heightCandidates
  .filter(v => Number.isFinite(v) && v >= 50 && v <= 200)[0]; // typical base 70–150 mm

// tablet-based defaults if missing
const defaultOverhangEachSideMm = 25;     // small, like in your screenshot
const baseLengthMm = Number.isFinite(lengthMm)
  ? lengthMm
  : (tabletWidthMm + 2 * defaultOverhangEachSideMm);

const baseHeightMm = Number.isFinite(heightMm) ? heightMm : 90; // looks closer to ~90 mm in your shot

// convert to overlay px
const baseWidthPx  = Math.round(baseLengthMm * pxPerMm);
const baseHeightPx = Math.round(baseHeightMm * pxPerMm);

// clamp: never more than +6% wider than the tablet (matches the original)
const maxWidthPx = Math.round(stoneW * 1.06);
const finalBaseW = Math.min(baseWidthPx, maxWidthPx);

// gentle tuck under the tablet
const overlapPx = Math.round(baseHeightPx * 0.25);

// centered, under tablet bottom
const baseRowStyle: React.CSSProperties = {
  position: 'absolute',
  left: 0,
  top: Math.max(0, Math.min(overlayH - baseHeightPx, stoneBottom - overlapPx)),
  width: overlayW,
  height: baseHeightPx,
  display: 'flex',
  justifyContent: 'center',
  alignItems: 'flex-start',
  zIndex: 25,
  pointerEvents: 'none'
};

const baseStyle: React.CSSProperties = {
  width: finalBaseW,
  height: baseHeightPx,
  backgroundImage: `url(${baseTextureUrl})`,
  backgroundSize: 'cover',
  backgroundPosition: 'center'
};
```

### Why this fixes “too wide”

* We now **trust the JSON** for base mm when present (length/width fields ≥300 mm).
* If the JSON has no explicit base length, the default overhang is **25 mm/side** (your screenshot scale), and we **cap** at +6% of the tablet width so it can’t sprawl.
