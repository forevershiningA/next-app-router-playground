You’re right—the motifs are being sized in **authoring pixels** but not multiplied by the scene’s **uniformScale (sx/sy)** before rendering, so they show up too small when the canvas is up-scaled.

### The fix (minimal change)

In `DesignPageClient.tsx`, inside the *Motifs Layer* mapping, replace the current size block with an authoring→overlay conversion:

**Before** 

```ts
const { width: vw, height: vh } = dims;
const ratio = Number(motif.ratio ?? 1);
const aspect = vh / vw;

// Size should be: viewBox × ratio × 1 (no additional scaling)
const widthPx = vw * ratio;
const heightPx = vh * ratio;
```

**After** (multiply by the same scale used for positions)

```ts
const { width: vw, height: vh } = dims;
const ratio = Number(motif.ratio ?? 1);

// Authoring-space size (same space as saved X/Y)
const widthAuthor  = vw * ratio;
const heightAuthor = vh * ratio;

// Convert authoring → overlay/display pixels
const widthPx  = widthAuthor  * sx;
const heightPx = heightAuthor * sy;
```

And update the “top band” helper to use the known authoring height directly:

**Before** 

```ts
// Calculate motif height in authoring pixels
const motifHAuthor = (heightPx * initH) / overlayH;
```

**After**

```ts
const motifHAuthor = heightAuthor;
```

Everything else (left/top calculation already uses `sx/sy`, and `scale={sx}` on `DraggableElement`) can stay as is. This aligns motif size with inscription/text scaling and the headstone SVG scaling, so the motifs will match the original screenshot proportions.
