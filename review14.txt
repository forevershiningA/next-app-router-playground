You’ve still got mixed-coordinate saves in the wild. Some designs were saved in **logical (authoring) px**, others in **physical px (DPR-scaled)**.
Forcing `raw / savedDpr` for everything will misplace items that were already logical. Detect the space first, then normalize **only when needed** (and do the same for font/motif sizes).

Drop-in patch (uses the helper already in your file):

```ts
// 1) Decide coordinate space per-design
const headstone = designData.find((i:any)=> i.type==='Headstone' || i.type==='HeadStone') || {};
const initW = headstone?.init_width || 707;
const initH = headstone?.init_height || 476;
const savedDpr = Number(headstone?.dpr) || 1;

// use the helper you already declared at top of file
const usesPhysical = designUsesPhysicalCoords(
  designData.filter((i:any)=> i.type==='Inscription' || i.type==='Motif'),
  initW, initH
);

// 2) Normalizers
const normXY = (rawX:number, rawY:number) => {
  // Convert center-origin to canvas if needed elsewhere in your pipeline
  const x = usesPhysical ? rawX / savedDpr : rawX;
  const y = usesPhysical ? rawY / savedDpr : rawY;
  return { x, y };
};

const normFont = (raw:number) => usesPhysical ? raw / savedDpr : raw;

// Motif width/height saved in authoring px? Some are DPR-scaled on iPhone.
// Normalize both if present.
const normSize = (w?:number, h?:number) => ({
  w: w != null ? (usesPhysical ? w / savedDpr : w) : undefined,
  h: h != null ? (usesPhysical ? h / savedDpr : h) : undefined
});

// 3) Apply everywhere you map items to the canvas
inscriptions.map((ins:any) => {
  const rawX = ins.x ?? ins.cx ?? 0;
  const rawY = ins.y ?? ins.cy ?? 0;
  const { x, y } = normXY(rawX, rawY);
  const fontSize = normFont(ins.fontSize || ins.size || 24);

  return {
    ...ins,
    // keep your center→top-left conversion after this if applicable
    canvasX: x,
    canvasY: y,
    fontSize,
  };
});

motifs.map((m:any) => {
  const rawX = m.x ?? m.cx ?? 0;
  const rawY = m.y ?? m.cy ?? 0;
  const { x, y } = normXY(rawX, rawY);
  const { w, h } = normSize(m.width, m.height);

  return {
    ...m,
    canvasX: x,
    canvasY: y,
    width: w,
    height: h,
  };
});

// 4) Helpful log
console.log('Coord space:', {
  usesPhysical, savedDpr, initW, initH
});
```

Why this fixes your screenshot: the SEHOVIC design was likely saved on an iPhone (DPR=3). If its coordinates were already logical, dividing again shrinks/offsets; if they were physical, **only then** do we divide. Same rule for font sizes and motif dimensions.

Tip: keep `savedDpr` on the **Headstone** object (fallback to 1), not per-item, and guard undefined sizes. Apply your crop offset and uniform scaling **after** this normalization.

This integrates cleanly with your current `DesignPageClient.tsx` (you already have `designUsesPhysicalCoords` defined at the top).
