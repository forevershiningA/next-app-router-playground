This is a complex coordinate geometry problem caused by legacy data formats (Physical vs. Logical pixels) and the mixing of Screenshot dimensions (which might be cropped) with JSON Coordinate dimensions (which are absolute to a canvas).

The main reason your design is breaking on mobile and sizing incorrectly on desktop is that **the code is trying to fit the JSON coordinates (which belong to the `init_width` × `init_height` canvas) into the `cropBounds` (screenshot) dimensions.**

To fix this, we need to:
1.  Establish the **Design Canvas** as the "Truth" (using `init_width` and `init_height` from the JSON).
2.  Ignore the screenshot crop for *positioning* logic.
3.  Implement a **Responsive Scale Factor** that simply shrinks that logical canvas to fit the user's screen (Desktop or Mobile).

### Why this works for iPhone 14 & Desktop:

1.  **Responsive Base:** We calculate `displayWidth` based on the window width (`viewportWidth * 0.92` for mobile). This ensures the canvas never overflows the phone screen.
2.  **Uniform Scale:** We calculate `uniformScale` as the ratio between the *Screen Size* and the *Original JSON Canvas Size*.
3.  **Coordinate Translation:**
    *   `offsetX` is exactly half the screen-size canvas.
    *   `rawX * uniformScale` converts the JSON coordinate (distance from center) to the Screen pixel distance.
4.  **DPR Handling:** By checking for `usesPhysicalCoords` inside the scaling logic and dividing by DPR *before* calculating the visual scale, we ensure that legacy "Retina" saves are normalized to logical pixels before being rendered.
