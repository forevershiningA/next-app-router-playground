Totally doable. Center it without fighting absolute `left` math—use a full-width “row” anchored at the stone’s bottom, then center the base inside that row with flex.

### Drop-in patch (inside the same overlay wrapper as the tablet)

(That wrapper must be `position: relative`.)

```tsx
// --- overlay scales (same as tablet)
const sx = overlayW / initW;
const sy = overlayH / initH;

// --- fitted tablet bounds (overlay px)
const stoneLeft = offsetX + (topProfile?.offX ?? 0) * sx;
const stoneTop  = offsetY + (topProfile?.offY ?? 0) * sy;
const stoneW    = (topProfile?.drawW ?? initW) * sx;
const stoneH    = (topProfile?.drawH ?? initH) * sy;
const stoneBottom = stoneTop + stoneH;

// --- base sizing
const baseOverhang = 0.04;
const baseWidth    = Math.round(stoneW * (1 + baseOverhang));
const baseHeight   = Math.round(overlayH * 0.085);
const overlapPx    = Math.round(baseHeight * 0.25); // tuck under tablet

// --- full-width row anchored at the stone bottom, then lift by overlap
const baseRowStyle: React.CSSProperties = {
  position: 'absolute',
  left: 0,
  top: Math.max(0, Math.min(overlayH - baseHeight, stoneBottom - overlapPx)),
  width: overlayW,
  height: baseHeight,
  display: 'flex',
  justifyContent: 'center',   // <-- centers the base
  alignItems: 'flex-start',
  zIndex: 25,
  pointerEvents: 'none',
};

// --- the actual base block
const baseStyle: React.CSSProperties = {
  width: baseWidth,
  height: baseHeight,
  backgroundImage: `url(${baseTextureUrl})`,
  backgroundSize: 'cover',
  backgroundPosition: 'center',
  backgroundColor: '#2b2b2b',
};

{/* Render AFTER the tablet background, INSIDE svgHostRef wrapper */}
<div style={baseRowStyle}>
  <div style={baseStyle} />
</div>
```

Why this works:

* The **row** spans the whole overlay, so there’s no ambiguous `left`.
* Flex **centers** the base horizontally under the stone.
* `top: stoneBottom - overlapPx` places it directly under the tablet with the desired overlap.

If the strip still floats mid-canvas, the base is not inside the same positioned wrapper as the tablet. Ensure the wrapper that contains the tablet has `position:'relative'` and this block is a **child** of it. The rest of your file structure supports this change.
